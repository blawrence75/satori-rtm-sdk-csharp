Interface IRtmClient

The IRtmClient interface is the main entry point for accessing the RTM service, including publish and subscribe operations.

Inherited Members
IDispatchObject.Dispatcher
Namespace:Satori.Rtm.Client
Assembly:Satori.RTM.Net45.dll
Syntax
public interface IRtmClient : IDispatchObject
Remarks

Create an instance of the client with RtmClientBuilder and use the IRtmClient methods to start, stop, and restart the client WebSocket connection, and publish and subscribe to channels. This snippet shows how to create a client: 

var client = new RtmClientBuilder("YOUR RTM ENFDPOINT", "YOUR APP KEY").Build();
await client.Start();
This snippet shows how to subscribe to a channel: 
var observer = new SubscriptionObserver();
observer.OnSubscriptionData += (sub, data) => 
    {
        foreach (var m in data.Messages)
            Console.WriteLine(m);
    };
await client.CreateSubscription(channel, SubscriptionModes.Simple, observer);
This snippet shows how to publish to a channel: 
await client.Publish(channel, "Hello World!", Ack.Yes);

Disposes the client.

Declaration
Task Dispose()
Returns
Type Description
Task

The task completes when this call is executed on the Dispatcher dispatcher.

Remarks

Call Dispose() when you are finished using the IRtmClient. The Dispose() method leaves the IRtmClient in an unusable state. After calling Dispose(), you must release all references to the IRtmClient so the garbage collector can reclaim the memory that the IRtmClient was occupying.

Start()

Starts the client. Client tries to establish the WebSocket connection to the RTM service asynchronously.

Declaration
Task Start()
Returns
Type Description
Task

The task completes when this call is executed on the Dispatcher dispatcher.

Remarks

The SDK attempts to reconnect to the RTM service if the WebSocket connection fails for any reason. If you make any publish or subscribe requests while the WebSocket connection is not active, the SDK queues the requests and completes them when the connection is established or re-established. You can use the client events to define application functionality for when the application enters or leaves the connected state.

Stop()

Stops the client. The SDK tries to close the WebSocket connection asynchronously and does not start it again unless you call Start() .

Declaration
Task Stop()
Returns
Type Description
Task

The task completes when this call is executed on the Dispatcher dispatcher.

Remarks

Use this method to explicitly stop all interaction with the RTM service. You can use the client events to define application functionality for when the application enters or leaves the connected state.
